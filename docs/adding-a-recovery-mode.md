# Adding another Figure 3 recovery mode

The playground keeps Figures 1 and 2 fixed and changes only the final local correction shown in Figure 3.
The original smooth animation remains in `app.js`; additional modes are handled by `local-corrections.js` and `local-corrections.css`.

## 1. Add the selector

In `index.html`, add a button with a unique `data-recovery-mode` value:

```html
<button class="mode-button"
        type="button"
        data-recovery-mode="constraint"
        aria-pressed="false">
  Constraints
</button>
```

## 2. Add a Figure 3 panel

Add a matching panel. Keep it hidden by default unless it is the initial mode.

```html
<div data-recovery-panel="constraint" hidden>
  <canvas id="constraintCanvas" width="520" height="190"></canvas>
  <p class="caption">Explain the local costate-to-action map.</p>
</div>
```

Add a matching equation row when the model or correction formulas differ:

```html
<section class="equation-row"
         data-recovery-equations="constraint"
         hidden>
  <!-- equation cards -->
</section>
```

## 3. Implement the renderer

In `local-corrections.js`:

1. collect the new canvas and text elements;
2. define the mode-specific parameters and local correction map;
3. draw the Figure 3 geometry and any convergence panel;
4. extend `setMode` so the title, panel, and equation row switch together;
5. optionally map the mode to a URL query such as `?mode=constraints`.

The shared Play and Reset buttons can drive the added animation by attaching the same event listeners used by the transaction-cost mode.

## 4. Add styles

Place mode-specific styles in `local-corrections.css`. Reuse the existing colors and card dimensions so the third column does not jump when switching modes.

## 5. Preserve the conceptual contract

Every Figure 3 mode should answer the same question:

\[
\text{estimated local costate signal}
\quad\longrightarrow\quad
\text{current action or policy structure}.
\]

Good candidates include constrained projection, delayed-state correction, non-exponential discounting, and optimal-stopping intervention.

## 6. Distinguish computation from illustration

State explicitly whether a displayed curve is

- generated by a live numerical routine,
- loaded from an experiment, or
- a schematic educational trajectory.

This prevents the visualization from implying a numerical experiment that the static page does not actually run.
